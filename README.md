# ACT Alpha Evaluation

This repository contains testing and evaluation scripts for generating, building, testing, and benchmarking compiler backends which use alpha operators for e-graph extraction. We compare alpha e-graphs in backends generated by the ACT compiler.

## Setup and Evaluation Flow

The following steps describe how to run and test the alpha-node backends.

### 1. Clone the Repository

```bash
git clone --recursive https://github.com/nkgithub01/act-alpha-evaluation
```

### 2. Build Docker Environment

```bash
./docker/build.sh --rebuild
```

### 3. Build QKV and QKV_new (ACT original)
```bash
./scripts/build_backend.sh QKV
./scripts/build_backend.sh QKV_new
```

### 4. Test QKV and QKV_new
```bash
./tests/pytest.sh QKV original
./tests/pytest.sh QKV_new original
```

### 5. Generate, Build, and Test QKV Alpha backend

```bash
./scripts/generate.sh QKV.py
./scripts/build_backend.sh QKV_improved
./tests/pytest.sh QKV improved
```

### 6. Generate, Build, and Test QKV_new Alpha Backend

```bash
./scripts/generate.sh QKV_new.py
./scripts/build_backend.sh QKV_new_improved
./tests/pytest.sh QKV_new improved
```

### 7. Compile attention on all backends

```bash
./scripts/compile_hlo.sh QKV_new attention.hlo
./scripts/compile_hlo.sh QKV_new_improved attention.hlo
./scripts/compile_hlo.sh QKV attention.hlo
./scripts/compile_hlo.sh QKV_improved attention.hlo
```

## Repository Structure

### `QKV.py`, `QKV_new.py`

TAIDL ISA specifications used to generate the alpha/improved QKV backends.

### `cost-models/`

Cost model implementations copied into generated improved backends during generation.

### `docker/`

Dockerfiles and helper scripts for the reproducible ACT evaluation environment.

### `examples/`

Example QKV inputs, expected data files, and HLO kernels for manual experiments.

### `generators/`

ACT tool generators: backend generator code plus the oracle generator submodule.

### `kernels/`

Standalone HLO kernels used by the compile scripts.

### `scripts/`

Host-side and in-container wrappers for generation, backend builds, compilation, and test execution.

### `smoothe/`

SmoothE submodule used for extraction/cost-model integration.

### `taidl/`

TAIDL submodule used to define accelerator ISAs and drive ACT code generation.

### `targets/`

Checked-in ACT-generated baseline backends for `QKV` and `QKV_new`; improved targets are generated here.

### `tests/`

Pytest validation suite and generated oracle packages for original and improved backends.

Generated build outputs are written to `backends/` and compiled kernels to `asm/` when the scripts are run.
